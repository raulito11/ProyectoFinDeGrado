=============================================================
  CURLS PARA PROBAR EL BACKEND — ProyectoFinDeGrado
=============================================================

NOTA: usa la flag -c cookies.txt / -b cookies.txt para
mantener la sesión entre peticiones (curl guarda la cookie
de sesión PHP en ese archivo).

NOTA WINDOWS: en CMD las comillas simples no funcionan.
Usa siempre comillas dobles y escapa las internas con \"
Ejemplo correcto:  -d "{\"campo\":\"valor\"}"

-------------------------------------------------------------
AUTH
-------------------------------------------------------------

# Login correcto
curl -s -c cookies.txt -X POST http://localhost/ProyectoFinDeGrado/backend/auth/login.php -H "Content-Type: application/json" -d "{\"email\":\"admin@restaurante.com\",\"password\":\"1234\"}"

# Login con credenciales incorrectas
curl -s -c cookies.txt -X POST http://localhost/ProyectoFinDeGrado/backend/auth/login.php -H "Content-Type: application/json" -d "{\"email\":\"noexiste@test.com\",\"password\":\"mal\"}"

# Login sin datos (debe devolver error de validación)
curl -s -c cookies.txt -X POST http://localhost/ProyectoFinDeGrado/backend/auth/login.php -H "Content-Type: application/json" -d "{}"

# Comprobar sesión activa (ejecutar después del login)
curl -s -b cookies.txt http://localhost/ProyectoFinDeGrado/backend/auth/session_check.php

# Comprobar sesión sin cookie (debe devolver 401)
curl -s http://localhost/ProyectoFinDeGrado/backend/auth/session_check.php

# Logout
curl -s -b cookies.txt -X POST http://localhost/ProyectoFinDeGrado/backend/auth/logout.php

# Comprobar sesión después del logout (debe devolver 401)
curl -s -b cookies.txt http://localhost/ProyectoFinDeGrado/backend/auth/session_check.php
